Settings

Configure performance and behavior options for the AI API Middleware

Performance Settings

Response Streaming

i
Enables streaming responses from the upstream API, reducing memory usage and improving response times for large content. When enabled, responses are sent in chunks rather than buffered entirely.

Stream responses for lower memory usage and reduced latency

Connection Pooling

i
Reuses HTTP connections across requests to improve performance and reduce connection overhead. Uses a session-based connection pool when enabled.

Reuse HTTP connections for better performance

Model Discovery Caching

i
Caches responses from model list API calls to reduce upstream requests. Cached responses are stored for the configured timeout period.

Cache model list responses for configurable time

Cache Timeout (seconds)

i
How long to cache model list responses before fetching fresh data from the upstream API. Longer timeouts reduce API calls but may serve stale data.

How long to cache model list responses

Model Cache Request Timeout (seconds)

i
Timeout for individual model cache API requests. This is separate from the cache duration and applies to the network request itself.

Timeout for model cache API requests

Request Settings

Maximum Retry Attempts

i
Number of times to retry failed requests with different API keys before giving up. Higher values increase reliability but may slow down responses.

Number of times to retry failed requests with different API keys

Request Timeout (seconds)

i
Maximum time to wait for a complete response from the upstream API. This includes connection time and data transfer time.

Maximum time to wait for a response from upstream API

Connect Timeout (seconds)

i
Maximum time to establish a connection to the upstream API server. This is separate from the overall request timeout.

Maximum time to establish connection to upstream API

Read Timeout (seconds)

i
Maximum time to wait for data from the upstream API after connection is established. Used for both streaming and non-streaming responses.

Maximum time to wait for data from upstream API

Streaming Timeout (seconds)

i
Maximum time to wait for streaming data chunks. Applied per chunk and used in conjunction with read timeout for streaming responses.

Maximum time to wait for streaming data chunks

Connection & Retry Settings

Total Retry Attempts

i
Maximum number of retry attempts for failed requests. This includes the initial attempt plus all retries. Used in the retry configuration for the requests library.

Maximum number of retry attempts for failed requests

Retry Backoff Factor

i
Multiplier for exponential backoff between retries. Higher values create longer delays between retry attempts, helping to avoid overwhelming the upstream API.

Multiplier for exponential backoff between retries

Connection Pool Size

i
Number of connection pools to cache when using connection pooling. Each pool can maintain multiple connections to the same host.

Number of connection pools to cache

Max Pool Connections

i
Maximum number of connections to save in the connection pool. Controls how many simultaneous connections can be maintained to the upstream API.

Maximum number of connections to save in the pool

Max Stream Retries

i
Maximum retry attempts for individual streaming chunks that fail. Used by the stream_with_retry function to handle network interruptions during streaming.

Maximum retry attempts for streaming chunks

Chunk Retry Delay (seconds)

i
Initial delay before retrying failed streaming chunks. This delay increases exponentially with each retry attempt.

Initial delay before retrying failed streaming chunks

Logging & Monitoring

Enable Request Logging

i
Log all incoming requests for debugging purposes. Includes request method, path, headers, and timing information. Useful for troubleshooting but may impact performance.

Log all incoming requests for debugging

Log Level

i
Minimum level of logs to capture. DEBUG shows all messages, INFO shows informational and above, WARNING shows warnings and errors, ERROR shows only errors.

Minimum level of logs to capture

Enable Metrics Collection

i
Collect performance metrics and statistics about API usage, latency, token counts, and error rates. Metrics are used for monitoring and optimization.

Collect performance metrics and statistics

Enable Performance Logging

i
Log detailed performance metrics including request latency, token usage, and throughput statistics. Helps identify performance bottlenecks.

Log detailed performance metrics

Log Request Body

i
Include request payloads in logs. Useful for debugging API calls but may log sensitive information. Only enable in development environments.

Include request payloads in logs

Log Response Body

i
Include response payloads in logs. Useful for debugging API responses but may log sensitive information. Only enable in development environments.

Include response payloads in logs

Rate Limiting

Enable Rate Limiting

i
Limit request rate per client to prevent abuse and ensure fair usage. When enabled, clients exceeding the rate limit will receive 429 responses.

Limit request rate per client

Requests Per Minute

i
Maximum requests allowed per minute per client IP address. This limit helps prevent abuse while allowing reasonable usage patterns.

Maximum requests allowed per minute per client

Rate Limiting Strategy

i
Algorithm for rate limiting: Fixed Window (resets at fixed intervals), Sliding Window (smooth rate limiting), Token Bucket (allows bursts).

Algorithm for rate limiting

Burst Allowance

i
Extra requests allowed in short bursts above the normal rate limit. Useful for handling legitimate burst traffic patterns.

Extra requests allowed in short bursts

Security

Enable CORS

i
Cross-Origin Resource Sharing support. Allows web applications from different domains to make requests to this API. Required for web-based clients.

Cross-Origin Resource Sharing support

CORS Origins

i
Allowed origins for CORS requests. Use '*' for all origins (less secure) or specify comma-separated domains like 'https://example.com,https://app.example.com'.

Allowed origins (comma-separated, * for all)

Max Request Size (bytes)

i
Maximum allowed request body size. Protects against oversized requests that could cause memory issues or be used for DoS attacks.

Maximum allowed request body size

Blocked User Agents

i
User agent strings to block. Requests from these user agents will be rejected. Useful for blocking unwanted bots or crawlers.

User agents to block (comma-separated)

Advanced Proxy Settings

Failover Strategy

i
Method for selecting backup API keys when the primary key fails. Round Robin cycles through keys sequentially, Random selects randomly, Least Used picks the least recently used key.

Method for selecting backup API keys

Default Buffer Size (bytes)

i
Default size of streaming buffer chunks for reading response data. Optimized automatically based on request size and content type, but this sets the baseline.

Default size of streaming buffer chunks

Small Request Threshold (bytes)

i
Request size below which smaller buffers are automatically used. Helps optimize memory usage for small requests like simple queries.

Request size below which smaller buffers are used

Large Request Threshold (bytes)

i
Request size above which larger buffers are automatically used. Optimizes for large requests like complex prompts or file uploads.

Request size above which larger buffers are used

Small Buffer Size (bytes)

i
Buffer size used for small requests. Smaller buffers reduce memory overhead for simple, lightweight requests.

Buffer size used for small requests

Large Buffer Size (bytes)

i
Buffer size used for large requests. Larger buffers improve throughput for big data transfers and streaming responses.

Buffer size used for large requests

Min Buffer Size (bytes)

i
Minimum allowed buffer size. Ensures buffers don't become too small to be efficient, preventing excessive system calls.

Minimum allowed buffer size

Max Buffer Size (bytes)

i
Maximum allowed buffer size. Prevents excessive memory usage that could impact system performance or stability.

Maximum allowed buffer size

JSON Buffer Limit (bytes)

i
Amount of streaming response to buffer for token extraction and usage statistics. Must be large enough to capture the JSON metadata.

Amount of streaming response to buffer for token extraction

Max Concurrent Requests

i
Maximum simultaneous requests allowed. Helps prevent resource exhaustion and ensures fair resource allocation across clients.

Maximum simultaneous requests allowed

Enable Request ID Injection

i
Add unique request IDs to all outgoing requests for tracing and debugging. Helps correlate requests across logs and monitoring systems.

Add unique request IDs for tracing
